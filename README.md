# <img src="https://github.com/Gracielleee/MobileDev_Group9/blob/main/app/src/main/res/drawable/unifime.png" width="32px" alt=""/> UnifiMe

**UnifiMe** is a personal CRM app designed to consolidate your contacts, notes, and reminders into one easy-to-use interface. More than just a contact management application, UnifiMe focuses on relationship management, enabling users to foster better connections.

</br>

***Disclaimer:***
This project was developed as part of a Mobile Development Fundamentals course with a focus on front-end development. 
It serves as a practical project to apply core Android app development concepts using Java and the Android SDK with Gradle, demonstrating the basic structure and functionality of a mobile application for educational purposes.

</br>

## ğŸš€ Quick Start

To get this mobile application up and running on your local development environment, follow these steps.

### Prerequisites
-   **Java Development Kit (JDK):** Version 11 or higher.
-   **Android Studio:** Latest stable version recommended.
-   **Android SDK:** Configured within Android Studio.
-   **An Android device or Emulator:** For running the application.

### Installation

1.  **Clone the repository**
    ```bash
    git clone https://github.com/Gracielleee/MobileDev_Group9.git
    cd MobileDev_Group9
    ```

2.  **Open in Android Studio**
    Launch Android Studio and select "Open an existing Android Studio project". Navigate to the cloned `MobileDev_Group9` directory and open it.

3.  **Sync Gradle Project**
    Android Studio should automatically detect the Gradle project and prompt you to sync. If not, click on the "Sync Project with Gradle Files" button (often a small elephant icon) in the toolbar. This will download all necessary dependencies.

4.  **Environment setup**
    For some Android projects, you might need to configure local properties or environment variables.
    -   **`local.properties`**: This file is typically generated by Android Studio and contains the path to your Android SDK. It is usually excluded from version control. If it's missing, Android Studio will prompt you to create it.
    -   **`gradle.properties`**: Review this file for any project-specific Gradle settings or properties that might need adjustment.

### Run Development Build

1.  **Select a target device/emulator**
    In Android Studio, select your desired Android Virtual Device (AVD) or a connected physical device from the dropdown menu in the toolbar.

2.  **Run the application**
    Click the "Run 'app'" button (green play icon) in the Android Studio toolbar. Android Studio will build the application and install it on your selected device/emulator.
